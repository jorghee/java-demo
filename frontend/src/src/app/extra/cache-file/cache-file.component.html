A simple cache using @Schedule

To update the table in the home page with the current status of the JSR we load a JSON file with the data.
The file is updated daily and stored in an external repository (e.g. AWS S3, Dropbox etc.).
To avoid an expensive communication with the external repository for each request we receive we store the JSON file in memory and we update it every 24h using @Schedule.
@Schedule is part of the Timer Service offered by the EBJ container and it allows you to create Programmatic and Automatic timers.
More informations here: Java Tutorial : Using the Timer Service
